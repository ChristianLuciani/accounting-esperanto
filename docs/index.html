Kontablo — A Graph-Based Universal Accounting Ontology for the M2M Agentic Economy

Kontablo is a graph-based, UUID-keyed universal accounting ontology that bridges local jurisdictional chart-of-accounts standards, international reporting frameworks (IFRS/XBRL), the agentic economy (MCP, A2A, AP2), and blockchain/DeFi protocols — enabling deterministic, machine-verifiable financial data exchange across all 195 sovereign jurisdictions, with statutory chart-of-accounts overlays for the 60 jurisdictions that mandate a national chart.

Author: Christian Luciani · ORCID 0000-0002-6955-5384 · Independent Researcher, Cuenca, Ecuador. Praxia (an Ecuadorian technology firm, in incorporation) is the initiative's planned commercial entity.

What problem does Kontablo solve?

Every major jurisdiction maintains its own chart of accounts. These standards share underlying economic concepts but differ in structure, naming, and hierarchy — the "Accounting Babel." Kontablo resolves them to a single UUID-keyed graph where economic nodes are jurisdiction-independent and local codes are metadata, not identity.

What is the Deterministic Boundary Library?

A set of hard logical constraints enforced before any agent decision. An agent cannot emit a UUID that does not exist in the ontology graph — this eliminates the classical class of accounting hallucination at the harness level and relocates residual error to the boundary of semantic coverage, which escalates to a human under the Co-responsibility Architecture.

Security posture

Threat horizon is multi-decade (financial records are long-lived), so the roadmap targets hybrid classical + post-quantum signatures using NIST FIPS 203 (ML-KEM) and NIST FIPS 204 (ML-DSA) on spec version hashes.
